# LLM 부서 배정 추론 지침

## 기본 원칙

부서의 내부 업무분장이 ibk_mapping_rules.md 또는 ibk-dept-mapping.md에 명시되지 않은 경우,
LLM은 다음 공개 기준을 순서대로 참조하여 추론한다.

추론 결과는 출력 JSON에 별도 필드로 표기하지 않는다.
내부 추론 근거만 활용하여 `dept` 및 `our_action`의 정확도를 높인다.

---

## 추론 참조 기준 (우선순위 순)

1. **금융위원회 「금융회사 지배구조 모범규준」**
   - 이사회, 감사위원회, 준법감시인, 내부통제위원회, 위험관리위원회의 역할 범위
   - 준법감시인 소관: 법규 준수 모니터링, 임직원 교육, 내부제보 처리
   - 위험관리책임자 소관: 리스크 한도 설정, 모니터링, BCP

2. **금융감독원 「내부통제 모범규준」**
   - 금융사고 예방·보고 라인: 검사부(감사) → 이사회 보고
   - 개인정보·정보보호: 정보보호최고책임자(CISO) 산하 부서 (IBK = 정보보호총괄부)
   - 소비자보호: 금융소비자보호총괄책임자(CCO) 산하 부서 (IBK = 금융소비자보호부)
   - AML: 보고책임자 산하 부서 (IBK = 자금세탁방지부)

3. **해당 법령의 금융업권별 적용 가이드라인**
   - 전자금융거래법 → IT내부통제부 / 개인디지털사업부
   - 신용정보법 → 정보보호총괄부 (신용정보관리·보호인 지정 의무)
   - 자본시장법 → 자산관리사업부 / 신탁부 / 사모투자부
   - 외국환거래법 → 외환사업부 / 글로벌사업부
   - 근로기준법·산업안전보건법 → 인사부

---

## 다중 부서 배정 기준

하나의 법령이 여러 부서에 영향을 줄 때:

- `dept`: 해당 법령의 **주 소관부서** 1개 (가장 직접적인 업무 변경 발생 부서)
- `related_depts`: 협조 부서 (대응 이행에 연관되나 주도하지 않는 부서)

판단 기준:
- 규정 개정 의무가 있는 내규의 소관부서 → `dept`
- 해당 내규를 참고·준수해야 하는 타 부서 → `related_depts`
- 고객 접점(영업) 부서는 통상 `related_depts` (내부통제 규정 직접 소관 아님)

## 절대 사용 금지 부서명

다음은 IBK에 존재하지 않는 명칭이므로 출력에 사용 금지:
법무실, 법무팀, 개인정보보호팀, AML팀, IT운영부, 경영전략부,
소비자보호부, 리스크관리팀, 준법감시팀, 컴플라이언스팀
