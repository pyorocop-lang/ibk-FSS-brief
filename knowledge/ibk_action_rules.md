# LLM 부서 배정 추론 지침

## 기본 원칙

부서의 내부 업무분장이 ibk_mapping_rules.md 또는 ibk-dept-mapping.md에 명시되지 않은 경우,
LLM은 다음 공개 기준을 순서대로 참조하여 추론한다.

추론 결과는 출력 JSON에 별도 필드로 표기하지 않는다.
내부 추론 근거만 활용하여 `dept` 및 `our_action`의 정확도를 높인다.

---

## 추론 참조 기준 (우선순위 순)

1. **금융위원회 「금융회사 지배구조 모범규준」**
   - 이사회, 감사위원회, 준법감시인, 내부통제위원회, 위험관리위원회의 역할 범위
   - 준법감시인 소관: 법규 준수 모니터링, 임직원 교육, 내부제보 처리
   - 위험관리책임자 소관: 리스크 한도 설정, 모니터링, BCP

2. **금융감독원 「내부통제 모범규준」**
   - 금융사고 예방·보고 라인: 검사부(감사) → 이사회 보고
   - 개인정보·정보보호: 정보보호최고책임자(CISO) 산하 부서 (IBK = 정보보호총괄부)
   - 소비자보호: 금융소비자보호총괄책임자(CCO) 산하 부서 (IBK = 금융소비자보호부)
   - AML: 보고책임자 산하 부서 (IBK = 자금세탁방지부)

3. **해당 법령의 금융업권별 적용 가이드라인**
   - 전자금융거래법 → 개인디지털사업부 (주담당) / IT내부통제부 (보안 협조)
   - 신용정보법 → 정보보호총괄부 (신용정보관리·보호인 지정 의무)
   - 자본시장법 → 자산관리사업부 / 신탁부 / 사모투자부
   - 외국환거래법 → 외환사업부 / 글로벌사업부
   - 근로기준법·산업안전보건법 → 인사부

## 다중 부서 배정 기준

하나의 법령이 여러 부서에 영향을 줄 때:

- `dept`: 해당 법령의 **주 소관부서** 1개 (가장 직접적인 업무 변경 발생 부서)
- `related_depts`: 협조 부서 (대응 이행에 연관되나 주도하지 않는 부서)

판단 기준:
- 규정 개정 의무가 있는 내규의 소관부서 → `dept`
- 해당 내규를 참고·준수해야 하는 타 부서 → `related_depts`
- 고객 접점(영업) 부서는 통상 `related_depts` (내부통제 규정 직접 소관 아님)

### 한 줄 판별법 (가장 중요)

> **"이번 법령 때문에 내규(규정·지침)를 직접 고쳐 써야 하는 부서가 `dept`,
> 그 고쳐진 내규를 따라야 하는 부서가 `related_depts`."**

비유: `dept`는 **시험문제를 출제하는 부서**(펜을 들고 내규를 개정), `related_depts`는 **그 문제로 시험 보는 부서**(개정된 내규를 준수·반영).
판단이 헷갈리면 다음 한 가지만 물어본다 — **"이 법령으로 어느 부서의 내규 조문이 바뀌는가?"** → 그 부서가 `dept`.

### 예시 (매핑 테이블 기준, 각 1개 dept + 협조부서)

| 외부 법령 | `dept` (주담당 = 내규 개정 주체) | `related_depts` (협조 = 개정 내규 준수) | 왜 이렇게 나뉘는가 |
|---|---|---|---|
| 신용정보법 개정 | **정보보호총괄부** — 「개인신용정보 관리·보호 지침」을 직접 개정 | 데이터혁신부 — 「마이데이터 업무지침」에 반영·준수 | 규정 개정 의무는 정보보호총괄부, 데이터혁신부는 그 기준을 따라 업무지침만 손봄 |
| 전자금융거래법 개정 | **개인디지털사업부** — 「전자금융규정」 소관, 직접 개정 | IT내부통제부 — 보안 통제 항목 준수·점검 | 규정 자체를 고치는 곳이 주담당, IT는 고쳐진 보안기준을 이행 |
| 금융소비자보호법 개정 | **금융소비자보호부** — 「금융소비자보호 규정」 직접 개정 | 여신기획부 — 대출상품 판매 접점에서 준수 | 여신기획부는 고객 접점(영업)이라 규정 직접 소관 아님 → 협조 |
| 특정금융정보법(특금법) 개정 | **자금세탁방지부** — AML 취급규정·지침 직접 개정 | (없음 → `[]`) | 단일 부서가 내규 개정·이행을 모두 전담 → 협조부서 근거 없으면 빈 배열 |

> 주의: `related_depts`는 **추론으로 만들지 않는다.** 위 예시처럼 `ibk_mapping_rules.md` ②테이블의 "IBK 소관부서" 컬럼에 실제로 함께 적힌 부서만 후보다. 근거가 없으면 빈 배열(`[]`)이 정답이다.

## 절대 사용 금지 부서명

다음은 IBK에 존재하지 않는 명칭이므로 출력에 사용 금지:
법무실, 법무팀, 개인정보보호팀, AML팀, IT운영부, 경영전략부,
소비자보호부, 리스크관리팀, 준법감시팀, 컴플라이언스팀
