## 2. 왜 만들었는가

### 기존 방식의 문제

| 항목 | 기존 수동 방식 |
|---|---|
| 소요 시간 | 금융위원회 홈페이지 접속 → 목록 확인 → 개별 열람 → 요약 작성 → 부서 배분 (약 1~2시간/일) |
| 누락 위험 | 담당자 부재·휴가 시 모니터링 공백 발생 |
| 일관성 | 담당자마다 중요도 판단·보고 형식이 달라짐 |
| 대응 속도 | 수집 후 검토까지 당일 오전 중반 이후 |

### AI 시스템 도입 후

| 항목 | 현재 자동화 방식 |
|---|---|
| 소요 시간 | 오전 7:30 자동 시작 → 7:55 보고서 완성 (25분, 무인) |
| 누락 위험 | 시스템 자동 실행 — 담당자 부재와 무관 |
| 일관성 | AI가 동일한 기준으로 분석, 표준 포맷 유지 |
| 대응 속도 | 출근 전 보고서 완성, 오전 업무 시작 즉시 검토 가능 |

## 3. 시스템이 하는 일 (5단계)

```
07:30  ① 수집     금융위원회 입법예고 전수 수집 (한국 IP, 자동)
                   → PDF 원문 포함 저장 (감사 대응)

07:32  ② 분석     Claude AI가 IBK 영향도 평가
                   → 🔴 즉시조치 / 🔶 중간검토 / 🔹 참고 등급 분류
                   → 담당 부서 자동 배정 (10개 부서)

07:40  ③ 보고서   DOCX 보고서 자동 생성
                   → 5개 섹션: 요약 / 법령브리핑 / 체크포인트 / 마감캘린더 / 용어해설

07:45  ④ 검증     AI 출력 품질 자동 검증 (10개 항목)
                   → 부서명 오류·글자 수 초과·형식 불일치 자동 감지

07:45  ⑤ 알림     Telegram으로 완료 알림 + 핵심 내용 즉시 전송
                   → 담당자가 이동 중에도 즉시 확인 가능
```

## 6. 운영 구조 (한 눈에)

```
[인터넷·클라우드]                    [사내 Windows PC]
  GitHub 서버                           내부통제점검팀 PC
  ┌─────────────────┐                 ┌──────────────────┐
  │  07:30 자동 발화  │──→ Telegram ──→│  크롤링 실행       │
  │  Claude AI 분석  │                 │  (금융위원회 접속) │
  │  DOCX 생성       │←── git push ───│  결과 전송         │
  │  알림 발송        │                 └──────────────────┘
  └─────────────────┘
         │
         ▼
  [결과물 자동 저장]
  • Telegram 알림 → 담당자 스마트폰
  • DOCX 보고서  → 로컬 PC + GitHub 90일 보관
  • PDF 원문     → 로컬 PC (감사 대응)
```

> **PC가 꺼져 있어도** 클라우드(GitHub)에서 분석·알림을 자동 실행합니다.  
> **크롤링만** 금융위원회의 해외 IP 차단 정책으로 인해 사내 PC에서 실행됩니다.
