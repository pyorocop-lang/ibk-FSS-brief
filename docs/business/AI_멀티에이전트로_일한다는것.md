# AI 멀티 에이전트로 일한다는 것 — IBK 내부통제 제재·경영유의 모니터링 사례

> **한 줄 요약**
> 매일 사람이 하던 "금융감독원 제재·경영유의 확인 → IBK 유사업무 자가점검 정리"를, **여러 AI 에이전트가 분업해 무인으로** 수행합니다.
> 사람은 *판단·검토*에 집중하고, 반복·수집·정리는 에이전트가 맡습니다.

## 1. 왜 AI에게 이 일을 맡겼나

금융감독원은 금융회사·임직원에 대한 **제재공시**와 검사 결과 **경영유의·개선사항**을 부정기적으로 게시합니다. 타 금융회사의 실제 제재사례는 IBK가 **동일한 통제 미비점을 선제적으로 자가점검**할 수 있는 가장 구체적인 벤치마킹 자료입니다. 기존에는 담당자가 **홈페이지 2곳(제재공시·경영유의) 접속 → 목록 확인 → 개별 열람(PDF) → 요약 → IBK 유사업무·담당 부서 연결**을 손으로 했습니다.

| 기존 수작업의 문제 | 결과 |
|---|---|
| 게시가 부정기적이라 매일 확인 필요 | 담당자 시간 소모·피로 |
| 담당자 부재·휴가 시 공백 | 신규 제재 모니터링 누락 위험 |
| 사람마다 IBK 연관성·중요도 판단이 다름 | 일관성 저하 |
| 타행 제재를 IBK 점검으로 연결하기까지 지연 | 대응 지연 |

→ 이 일은 **반복적이고, 규칙이 분명하고, 누락되면 안 되는** 일입니다. AI 에이전트가 가장 잘하는 영역입니다.

## 2. "AI 멀티 에이전트"란? (쉬운 비유)

하나의 만능 비서에게 모든 걸 시키는 대신, **각 분야 전문가로 구성된 작은 팀**을 두고 일을 **이어달리기(릴레이)**처럼 넘기는 방식입니다.

```
🕵️ 수집 담당  →  🧠 분석 담당  →  ✍️ 보고서 담당  →  ✅ 검토 담당  →  🗂️ 기록 담당  →  📨 알림 담당
 (자료 모으기)   (AI가 IBK 영향 판단)  (읽기 쉽게 작성)    (품질 점검)      (감사 보관)     (담당자 통지)
```

- 각 담당(에이전트)은 **한 가지 일만** 잘합니다. → 문제가 생겨도 어디서 났는지 명확하고, 한 명이 실수해도 전체가 무너지지 않습니다.
- 이 중 **"분석 담당"만 진짜 AI(LLM, Claude Haiku)** 이고, 나머지는 정해진 규칙대로 정확히 움직이는 자동화입니다.
  → "AI에게 다 맡긴다"가 아니라, **AI가 가장 잘하는 판단 한 곳에만** 쓰고 나머지는 검증 가능한 규칙으로 두는 것 — 이게 신뢰성 중심 설계의 핵심입니다.

## 3. 우리 업무에 적용한 모습 (하루 흐름)

매일 **오전 08:00 · 오후 16:00**(2회), 노트북이 꺼져 있어도 클라우드에서 자동으로 돌아갑니다.

1. **🕵️ 수집** — 금융감독원 홈페이지의 **제재공시(HTML)와 경영유의·개선(PDF)** 2개 게시판을 직접 훑습니다(제재 관련 공개 API가 없어 HTML/PDF를 직접 스크래핑). **게시일 기준일(앵커)과 중복방지 원장**으로 *진짜 새로 게시된 건만* 골라냅니다. (과거에 계속 떠 있는 오래된 공시는 제외 — '새 것'만 다룸)
2. **🧠 분석(AI)** — Claude AI가 각 제재사례가 **IBK에도 발생 가능한지** 평가하고, **기관 계층(T0 IBK / T1 은행 / T2 인접금융 / T3 주변)**과 **위험도(🔴 상 / 🔶 중 / 🔹 하)** 로 등급을 매기고, **담당 부서를 자동 배정**합니다. *타행 제재 → IBK 자가점검 제안 / IBK 직접 제재 → 즉시 대응.*
3. **✍️ 보고서** — 각 건을 **제재대상 → 무슨 일이 있었나요? → IBK에도 발생 가능한가요? → 무엇을 점검할까요?** 카드형 문서(Word)로 자동 생성합니다.
4. **✅ 검토** — AI가 쓴 글의 품질을 **다른 자동 점검기**가 검사합니다. (부서명 오류·문장 길이·금지 표현·보고서 구조 누락 등)
5. **🗂️ 기록** — 모든 실행을 감사 추적이 가능하도록 보관합니다(원본 HTML·PDF·메타·로그, 그리고 *신규가 없는 날에도* '무엇을 스캔했는지'를 남기는 스캔 증적).
6. **📨 알림** — 담당자 휴대폰으로 핵심 내용을 즉시 전송합니다. (오후 알림은 *오전 이후 새로 생긴 건이 있을 때만* — 불필요한 알림을 줄임)

> 전 과정 약 **2~4분**. 담당자는 출근 전 완성된 보고서를 받아 **검토와 의사결정**부터 시작합니다.

## 4. 사람과 AI의 역할 분담 (Human-in-the-loop)

AI가 모든 걸 결정하지 않습니다. **사람의 감독**이 설계에 내장돼 있습니다. 제재는 법적으로 민감하므로 AI는 단정하지 않고 **"점검 제안"형**으로만 말합니다.

| AI 에이전트가 하는 일 | 사람(담당자)이 하는 일 |
|---|---|
| 수집·요약·등급 초안·부서 배정·문서 작성 | 보고서 **검토**, 실제 **대응·점검 의사결정** |
| 품질 자동 점검(가드레일) | 경고 표시된 항목 **확인·보완** |
| 하루 2회 무인 반복 | 정책 판단, 우선순위 결정 |

→ AI는 **준비를 끝내주고**, 사람은 **판단에 집중**합니다.

## 5. "AI가 틀리면 어쩌나?" — 신뢰를 확보한 4가지 장치

AI 자동화의 가장 큰 우려는 *"틀린 걸 맞다고 보고하면?"* 입니다. 이를 막기 위해:

1. **검토 에이전트(가드레일)** — AI가 작성한 내용을 또 다른 자동 점검기가 검사합니다. 사람이 매번 전수 검수하지 않아도 오류를 자동 포착합니다.
2. **"모르면 멈춘다" 원칙** — 자료 수집이 실패하면 *"신규 없음"으로 둘러대지 않고* 명확히 **실패로 처리**하고(성공 기록은 비파괴 보존), 재실행을 유도합니다. (데이터 미확인을 안전으로 오인하지 않음)
3. **추정은 숨기지 않고 표시** — AI 분석이 불가한 상황(예: API 오류)에서 규칙 기반으로 채운 항목은 *"(키워드 추정 — 검토 필요)"* 라벨을 붙여 사람이 알아볼 수 있게 합니다.
4. **완전한 감사 추적** — 모든 실행·원본·판단 근거를 기록으로 남깁니다. **신규가 없는 날에도** '그날 무엇을 스캔했는지'(목록 전건 + 원본 지문)를 영구 보존해, *왜 이렇게 보고됐는지*와 *실제로 확인은 했는지*를 사후에 추적할 수 있습니다. (내부통제·감사 대응 필수)

| 비용 | 외부 솔루션 비용 없이 **AI 사용료(종량제)만** 발생 |

| **컨텍스트 엔지니어링**(지식 주입) | 조직도·부서매핑·규칙·톤을 코드와 분리해 AI에 주입 |

| **휴먼 인 더 루프**(사람 감독) | 사람은 검토·의사결정, AI는 준비 |
| **거버넌스/감사 추적** | 모든 실행 기록·추적 가능(신규 0건인 날도 스캔 증적) |

> 핵심 메시지: **"AI에게 다 맡긴다"가 아니라, AI를 가장 잘하는 곳에 쓰고 사람이 통제·검증하는 협업 체계.**

## 8. 한계와 향후

- 현재 AI는 *초안·분류·작성*을 담당하며, **최종 판단은 사람**이 합니다(의도된 설계).
- 모니터링 대상은 금융감독원 제재·경영유의입니다. 향후 타 기관·추가 알림 채널 등을 검토할 수 있습니다.
- **인프라 특성:** 자매 프로젝트(FSC 법령 브리핑)와 달리, 금융감독원 사이트는 **해외 클라우드에서도 접속이 막히지 않음이 검증**되어(미국 러너 접근 PASS) 별도 우회(프록시) 없이 직접 수집합니다 — 구조가 단순하고 장애 지점이 적습니다.
- **"자동 감시가 실제로 작동한 사례":** 2026-07 초, 브리핑이 '오늘 신규'라며 게시일이 며칠 전인 오래된 제재를 함께 노출하던 문제가 있었습니다. 이를 **가상 시나리오 테스트로 재현·확인**해 *게시일 기준일(앵커)+중복방지 원장* 방식으로 개선했고, 같은 시기 **감사 증적(scanAudit)**을 추가해 신규가 없는 날에도 '무엇을 스캔했는지'를 영구 보존하도록 강화했습니다 — 문제를 스스로 잡아 개선·증적까지 이어진 사례입니다. (상세: [최근 개선사항](../deliverables/06_개선사항_2026-07.md))
- 남은 개선은 지속 과제로 두되, 현재도 정상 운영됩니다.
