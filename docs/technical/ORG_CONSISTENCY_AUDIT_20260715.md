## 독립 교차검증 후 보완

Claude 교차검증에서 `검사부`가 ASCII 조직도에만 있고 레지스트리가 읽는 Markdown 불릿에는 없어, 내부통제 fallback의 협조부서 검증이 실패하는 회귀가 확인됐다. 다음과 같이 보완했다.

- 감사 직속 현행 조직 `검사부`를 파싱 가능한 불릿으로도 명시해 레지스트리를 94개로 완성
- 사용자 확인 매핑만 점검하던 테스트를 기본값까지 포함한 fallback 전체 18개 경로로 확대
- 분석 결과 검증 예외를 정상 fallback 종료코드 1과 구분해 치명 오류 2로 중단
- 현행·전환 구간 경계가 사라진 비정상 조직 정본의 파싱 거부를 회귀 테스트로 고정

과거 산출물에서는 `legacy_dept`가 원래 **부서명**만 보존한다. 함께 정정된 생성 서술의 이전 버전은 현행 JSON에 중복 보존하지 않으며 Git 이력에서 확인할 수 있다.

## 변경하지 않은 기록

금감원 원문을 보존한 `bodyText`에는 타 금융기관의 `재무회계부` 등 부서명이 포함될 수 있다. 이는 IBK 부서 배정값이 아니라 원문 증적이므로 수정하지 않았다. 조직 정합성 검증 대상은 생성 필드인 `dept`, `related_depts`, `our_action`, `ctrl_insight`, `tgMsg`다.

## 자동 검증

- `org_registry.js`: 활성 반기 JSON 정본에서 안정 ID·현행 부서명 집합 생성
- `analyst.js`: LLM·fallback 결과 저장 전 현행 조직명 확인
- `validator.js` B5: 보고서 데이터의 주담당·협조부서 재검증
- `test/org_consistency.test.js`: fallback 전체 경로, 폐지부서 제외, 정본 경계, 프롬프트 상충, 비현행 부서 차단 회귀검증
